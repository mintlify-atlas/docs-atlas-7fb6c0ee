---
title: "Virtual slides"
description: "Render only the slides that are visible on screen to keep Swiper fast with large or dynamic datasets."
keywords: ['swiper virtual slides', 'swiper performance', 'swiper large list', 'swiper virtual']
---

The Virtual module renders only the slides that are within or near the viewport, keeping the DOM lightweight regardless of how many total slides you have. This makes Swiper practical for datasets with hundreds or thousands of items.

<CardGroup cols={2}>
  <Card title="Performance" icon="bolt">
    Only a small subset of DOM nodes exist at any time, reducing memory use and layout cost.
  </Card>
  <Card title="Dynamic updates" icon="arrows-rotate">
    Add, prepend, or remove slides at runtime without reinitializing the swiper.
  </Card>
</CardGroup>

---

## Setup

```js
import Swiper from 'swiper';
import { Virtual } from 'swiper/modules';
import 'swiper/css';
```

Pass your full slides array to `virtual.slides`. Swiper calculates which subset to render based on the active index and the `addSlidesBefore` / `addSlidesAfter` buffers.

```js
const slides = Array.from({ length: 1000 }, (_, i) => `Slide ${i + 1}`);

const swiper = new Swiper('.swiper', {
  modules: [Virtual],
  virtual: {
    slides,
  },
});
```

### HTML structure

The HTML structure is identical to a standard Swiper. Virtual slides are injected dynamically into `.swiper-wrapper`:

```html
<div class="swiper">
  <div class="swiper-wrapper">
    <!-- Swiper inserts virtual slides here -->
  </div>
</div>
```

## Options

| Option | Type | Default | Description |
|---|---|---|---|
| `enabled` | `boolean` | `false` | Enable virtual slides. |
| `slides` | `any[]` | `[]` | Array of slide data items. Each item is passed to `renderSlide`. |
| `cache` | `boolean` | `true` | Cache rendered slide HTML elements. Cached elements are reused instead of re-rendered. |
| `addSlidesBefore` | `number` | `0` | Number of extra slides to pre-render before the visible range. |
| `addSlidesAfter` | `number` | `0` | Number of extra slides to pre-render after the visible range. |
| `slidesPerViewAutoSlideSize` | `number` | `320` | Assumed slide size (in px) when `slidesPerView: 'auto'` is used. |
| `renderSlide` | `function \| null` | `null` | `(slide, index) => string \| HTMLElement` — returns the outer HTML of a single slide. |
| `renderExternal` | `function \| null` | `null` | `(data: VirtualData) => void` — delegates rendering to an external library (React, Vue, etc.). |
| `renderExternalUpdate` | `boolean` | `true` | Update Swiper layout immediately after `renderExternal` is called. Set to `false` if your renderer is asynchronous. |

### Custom slide rendering

Use `renderSlide` to control each slide's markup:

```js
const swiper = new Swiper('.swiper', {
  modules: [Virtual],
  virtual: {
    slides: products,
    renderSlide(product, index) {
      return `
        <div class="swiper-slide">
          <img src="${product.image}" alt="${product.name}" />
          <p>${product.name}</p>
        </div>
      `;
    },
  },
});
```

## Methods

Access virtual slide methods via `swiper.virtual`. These methods apply only to the **Core** version. In React and Vue, manage slides by updating your component's data source.

### `appendSlide(slide)`

Append one or more slides to the end of the virtual slides array.

```js
// Append a single slide
swiper.virtual.appendSlide('<div class="swiper-slide">New slide</div>');

// Append multiple slides
swiper.virtual.appendSlide([
  '<div class="swiper-slide">Slide A</div>',
  '<div class="swiper-slide">Slide B</div>',
]);
```

### `prependSlide(slide)`

Prepend one or more slides to the beginning of the virtual slides array.

```js
swiper.virtual.prependSlide('<div class="swiper-slide">First slide</div>');
```

### `removeSlide(slideIndexes)`

Remove one or more slides by index.

```js
swiper.virtual.removeSlide([0, 2]); // Remove slides at index 0 and 2
```

### `removeAllSlides()`

Remove every virtual slide.

```js
swiper.virtual.removeAllSlides();
```

### `update(force)`

Recalculate the rendered range. Pass `true` to force a full re-render.

```js
swiper.virtual.update(true);
```

## Properties

```js
swiper.virtual.slides;  // Full array of virtual slide items
swiper.virtual.cache;   // Object of cached slide HTMLElements keyed by index
swiper.virtual.from;    // Index of the first currently rendered slide
swiper.virtual.to;      // Index of the last currently rendered slide
```

## React usage

In React, use `virtual` with `virtualIndex` on each `<SwiperSlide>`. Manage the slides array in component state and let React handle re-renders.

```jsx
import { useState } from 'react';
import { Swiper, SwiperSlide } from 'swiper/react';
import { Virtual } from 'swiper/modules';
import 'swiper/css';

const slides = Array.from({ length: 500 }, (_, i) => `Slide ${i + 1}`);

export default function VirtualExample() {
  return (
    <Swiper
      modules={[Virtual]}
      virtual
      slidesPerView={3}
      spaceBetween={10}
    >
      {slides.map((slide, index) => (
        <SwiperSlide key={slide} virtualIndex={index}>
          {slide}
        </SwiperSlide>
      ))}
    </Swiper>
  );
}
```

<Note>
  The `virtualIndex` prop is required in React and Vue. It tells Swiper each slide's position in the full dataset so the virtual rendering logic can determine what to show.
</Note>

## Vue usage

```vue
<template>
  <Swiper
    :modules="[Virtual]"
    :virtual="true"
    :slides-per-view="3"
    :space-between="10"
  >
    <SwiperSlide
      v-for="(slide, index) in slides"
      :key="slide"
      :virtual-index="index"
    >
      {{ slide }}
    </SwiperSlide>
  </Swiper>
</template>

<script setup>
import { Swiper, SwiperSlide } from 'swiper/vue';
import { Virtual } from 'swiper/modules';
import 'swiper/css';

const slides = Array.from({ length: 500 }, (_, i) => `Slide ${i + 1}`);
</script>
```

## Performance considerations

<AccordionGroup>
  <Accordion title="When to use virtual slides">
    Use virtual slides when your slider contains more than ~50 slides. For smaller sets, the overhead of virtual rendering is unnecessary and the default DOM-based approach is simpler.
  </Accordion>
  <Accordion title="Buffer tuning">
    The default render window is the visible slides plus a small browser-determined buffer. Increase `addSlidesBefore` and `addSlidesAfter` if you notice slides appearing blank during fast swipes. Keep values small (1–3) to preserve the memory benefit.
  </Accordion>
  <Accordion title="Cache behavior">
    With `cache: true` (default), Swiper stores rendered slide elements and reuses them when a slide re-enters the viewport. Disable caching only if your slide content changes between renders.
  </Accordion>
  <Accordion title="Asynchronous rendering">
    If you use `renderExternal` with an async renderer (such as a React root that defers paint), set `renderExternalUpdate: false` and call `swiper.virtual.update()` manually after your renderer finishes.
  </Accordion>
</AccordionGroup>
